# 小程序隐私与提审准备占位说明

当前工程已经具备可联调的小程序首版代码，但正式上传与提审前，仍需要你在微信公众平台补齐以下内容：

## 1. 隐私保护指引

至少准备并填写：

- 收集的信息类型
  - 用户名
  - 昵称
  - 年龄
  - 性别
  - 身高
  - 体重
  - 目标体重
  - 目标睡眠
  - 饮食记录
  - 运动记录
  - 睡眠记录
- 使用目的
  - 用户身份识别
  - 健康记录展示
  - 目标与趋势分析
  - 用户资料维护

## 2. 服务内容描述

建议统一口径为：

- 日常健康记录
- 饮食、运动、睡眠、体重管理
- 每日目标与趋势查看

避免写成：

- 医疗诊断
- 治疗方案
- 疾病判断

## 3. 审核演示路径

建议准备一套固定截图或演示账号路径：

1. 登录
2. 首页查看今日状态
3. 饮食记录提交
4. 运动记录提交
5. 睡眠记录提交
6. 体重记录提交
7. 我的页修改资料

## 4. 当前代码层已完成但仍需你手动确认的事项

- 已有小程序工程目录
- 已有页面与接口接线
- 仍需替换真实 `AppID`
- 已填入当前联调用 `API_BASE`
- 仍需配置 `request合法域名`
- 仍需在微信开发者工具和真机各跑一遍
