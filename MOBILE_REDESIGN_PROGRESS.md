# 移动端独立重构进度

## 背景

- 现有桌面端布局在电脑浏览器中基本满足需求。
- 现有移动端体验已确认不适合继续用“桌面端缩放 + 局部补丁”的方式维护。
- 已决定：桌面端保持现状，移动端采用独立布局和独立视觉层级，但继续复用当前后端接口和前端业务逻辑。

## 已确认的决策

- 方向：重写移动端展示层，不重写后端。
- 桌面端：保持不动，避免引入回归。
- 视觉：参考苹果系高级简约感，当前已转为偏冷静的雾蓝绿 / 冰感浅灰配色，不再使用米白主调。
- 交互：更接近原生 App，弱化网页式大段首屏。
- 导航：采用底部 4 主导航方案。
  - 首页
  - 记录
  - 趋势
  - 我的
- 性能：移动端默认进入轻量模式，减少 blur、厚阴影、复杂动画和不必要的滚动跳转。
- 数据逻辑：继续复用桌面端现有接口、表单提交、图表渲染和后端规则，不改接口契约。

- 移动端仍在“高密度精修”阶段，尚未完全定稿。
- 首页虽然已改成独立结构，但仍需继续优化：
  - 指标区块之间仍要持续检查是否有重复信息
  - 信息优先级和单屏可视效率仍可继续推敲
- 趋势页虽然已压缩，但仍需继续验证：
  - 报告区的四张小卡是否还可以再省空间
  - 评估结果区的 BMI 分段图在不同手机宽度下是否完全显示
- 我的页虽然已压缩，但仍需继续验证：
  - 账户行在不同长度用户名下是否稳定保持单行
  - 设置区在更小屏幕下是否仍有多余留白
- 当前最需要的仍然是真机截图回路验证，而不是再扩大结构改动范围。
